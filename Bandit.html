I also tried looking up a solution for this level and found this snippet:

xxxxxxxxxx
find / -user bandit7 -group bandit6 -size 33c -type f 2>/dev/null

Here's what I understood about the command:

  • / -> Means the directory 'root'
  • -size 33c -> Means whatever we are looking for is 33 bytes in size
  • -type f -> Means we are looking for a file and not a link or a directory
  • 2>/dev/null -> Means all the errors and warnings are sent to the /dev/null which basically deletes them

About the 2>/dev/null, I learned it from Here:

The > operator redirects the output usually to a file but it can be to a device. You can also use >> to append.

If you don't specify a number then the standard output stream is assumed, but you can also redirect errors:

> file redirects stdout to file 1> file redirects stdout to file

2> file redirects stderr to file

&> file redirects stdout and stderr to file > file 2>&1 redirects stdout and stderr to file

/dev/null is the null device it takes any input you want and throws it away. It can be used to suppress any output.

Note that > file 2>&1 is an older syntax which still works, &> file is neater, but would not have worked on older systems.

I discovered solving this level that Caesar Encryption is also called a ROT or Rotation Cypher so ROT13 just means rotating the letters to +13 order which means A becomes N.

Easiest way to decrypt a ROT13 encryption that came to my mind was using the tr (translate) command. A simple search brought me to This Page in which the 8th Sub-topic says:

ROT (Caesar Cipher) is a type of cryptography wherein encoding is done by moving the letters in the alphabet to its next letter.

Let's check how to use tr for encrypting.

In the following example each character in the first set will be replaced with the corresponding character in the second set.

The first set specified is [a-z], which means abcdefghijklmnopqrstuvwxyz. The second is [n-za-m], which turns into pqrstuvwxyzabcdefghijklmn.

xxxxxxxxxx
tr 'a-z' 'p-za-n'

Simple command to shows the above theory:

xxxxxxxxxx
$ sudo echo 'abg' | tr 'ab' 'ef'
Output
efg

This is very useful when you need to encrypt an email. Check below

xxxxxxxxxx
$ sudo echo 'cryptography@example.com' | tr 'A-Za-z' 'N-ZA-Mn-za-m'

Output

xxxxxxxxxx
pelcgbtencul@rknzcyr.pbz

So, I quickly added up the values and used this command:

xxxxxxxxxx
cat data.txt | tr 'a-z' 'n-za-m' | tr 'A-Z' 'N-ZA-M'

It worked like charm. It was basically a ROT13 decryption.

Basically, you have to find the type of file, then decompress it, then find the filetype and then decompress again. Do that until you get to a file that is ASCII type.

This is the command used to revert from a data dump to binary:

xxxxxxxxxx
xxd -r data.txt data

And this command is used to check the type/content/category of file:

xxxxxxxxxx
file data

This will tell you that the file named 'data' is a gzip file. Now all you need to do is to rename the file and add the .gz extension to the file. Then extract the file using gzip after that you will get a new file and then you will repeat similar steps again and again. The only thing that changes is the filetype and how to extract it.

In case of gzip file:

xxxxxxxxxx
gzip -d filename.gz

In case of bzip file:

xxxxxxxxxx
bzip2 -d filename.bz

In case of tar file:

xxxxxxxxxx
tar -x -f filename.tar

After all the extraction and file type finding, we finally get to file named data8 without any extension. Using cat data8 finally gives us the output we want.
